## ექსპერიმენტები და არქიტექტურული გადაწყვეტილებები

### 1. SimpleCNN (საბაზისო მოდელი)
**არქიტექტურა:** 2 conv layer (8, 16 ფილტრი) → 2 FC layer (128 ნეირონი)

**გადაწყვეტილება:** დავიწყე ყველაზე მარტივი შესაძლო არქიტექტურით, რათა მეჩვენებინა underfitting-ი.

**შედეგი:** Train Acc: 43.6%, Val Acc: 49.5%

**ანალიზი:** Val სიზუსტე მაღალია Train-ზე, რაც მიუთითებს underfitting-ზე — მოდელი ძალიან მარტივია სახის გამომეტყველებების სირთულის შესასწავლად. მცირე რაოდენობის ფილტრები (8, 16) ნიშნავს, რომ მოდელს არ შეუძლია საკმარისი მახასიათებლების ამოღება.

---

### 2. MediumCNN
**არქიტექტურა:** 3 conv layer (32, 64, 128 ფილტრი) + BatchNorm + Dropout(0.5)

**გადაწყვეტილება:** დავამატე მეტი ფილტრი, BatchNorm სასწავლო სტაბილურობისთვის და Dropout overfitting-ის შესამცირებლად.

**შედეგი:** Train Acc: 50.4%, Val Acc: 57.2% (+7.7% SimpleCNN-თან შედარებით)

**ანალიზი:** მკაფიო გაუმჯობესება SimpleCNN-თან შედარებით. BatchNorm-მა დაასტაბილიზირა სწავლება და უფრო სწრაფად და-converge-და. Val სიზუსტე კვლავ მაღალია Train-ზე, ანუ მოდელი ოდნავ underfitting-ს განიცდის.

---

### 3. DeepCNN
**არქიტექტურა:** 4 conv block (32, 64, 128, 256 ფილტრი) + BatchNorm + Global Average Pooling

**გადაწყვეტილება:** დავამატე მეტი სიღრმე და Flatten შევცვალე Global Average Pooling-ით პარამეტრების შესამცირებლად.

**შედეგი:** Train Acc: 58.6% | Val Acc: 61.6% (+4.4% MediumCNN-თან შედარებით)

**ანალიზი:** საუკეთესო vanilla CNN. Train სიზუსტე უახლოვდება Val-ს — სხვაობა მცირდება. Global Average Pooling ეფექტური აღმოჩნდა overfitting-ის შესამცირებლად.

---

### 4. ჰიპერპარამეტრების ოპტიმიზაცია

#### სწავლის სიჩქარის შედარება (DeepCNN)
| LR | Val Acc | ანალიზი |
|----|---------|----------|
| 0.01 | 25.2% | **დივერგენცია** — ძალიან მაღალი, მოდელი შემთხვევით გამოცნობაზე დარჩა |
| 0.001 | 61.6% | **საუკეთესო** — სტაბილური კონვერგენცია |
| 0.0001 | 55.7% | **ძალიან ნელი** — მე-15 ეპოქაზეც სწავლობდა, მეტი epoch სჭირდება |

**დასკვნა:** სწავლის სიჩქარე ყველაზე კრიტიკული ჰიპერპარამეტრია. lr=0.01 სრულ დივერგენციას იწვევს Adam-თან, რაც ადასტურებს, რომ Adam მგრძნობიარეა მაღალი LR-ის მიმართ.

#### ოპტიმიზატორის შედარება (DeepCNN)
| ოპტიმიზატორი | Val Acc |
|--------------|---------|
| Adam (lr=0.001) | 61.6% |
| SGD (lr=0.01, momentum=0.9) | 60.4% |

**დასკვნა:** Adam ოდნავ სჯობს SGD-ს, თუმცა SGD momentum-ით კონკურენტუნარიანია.

### 5. CNN+LSTM
**არქიტექტურა:** CNN feature extractor → სურათი ჰორიზონტალურ ზოლებად → LSTM ამუშავებს ზოლებს თანმიმდევრობით → FC კლასიფიკატორი

**გადაწყვეტილება:** გამოვცადე სახის რეგიონების თანმიმდევრული დამუშავება (შუბლი → თვალები → ცხვირი → პირი).

**ჰიპერპარამეტრების ოპტიმიზაცია:**
| Run | Hidden | Layers | Val Acc |
|-----|--------|--------|---------|
| CNNLSTM-baseline | 256 | 2 | 60.6% |
| CNNLSTM-1layer | 256 | 1 | 61.3% |
| CNNLSTM-hidden512 | 512 | 2 | 58.5% |
| CNNLSTM-dropout0.5 | 256 | 2 | 60.5% |

**დასკვნა:** მარტივი LSTM (1 layer, 256 hidden) სჯობს უფრო რთულ კონფიგურაციებს. Hidden size-ის გაზრდა 512-მდე გამოიწვია overfitting. ეს აჩვენებს, რომ CNN-ის მახასიათებლები საკმარისად მდიდარია — LSTM-ს დიდი ზომა არ სჭირდება.

---

### 6. ResNet18 (Transfer Learning)
**არქიტექტურა:** წინასწარ გაწვრთნილი ResNet18, შეცვლილი პირველი conv layer-ით (1 channel) და FC layer-ით (7 კლასი)

**გადაწყვეტილება:** გამოვცადე ImageNet-ზე წინასწარ გაწვრთნილი წონები FER2013-ზე.

**შედეგი:** Val Acc: 59.4% (DeepCNN-ზე ცუდი)

**ანალიზი:** Transfer learning **არ დაგვეხმარა**. მიზეზები:
- წინასწარ გაწვრთნილი წონები RGB სურათებისთვისაა; conv1-ის გრეისქეილზე შეცვლა ადრეული layer-ების წონებს პრაქტიკულად შემთხვევით ხდის
- 48x48 სურათები ძალიან პატარაა ResNet18-ისთვის, რომელიც 224x224-ისთვის არის.

**დასკვნა:** Transfer learning ყოველთვის სასარგებლო არ არის.

---

### 7. Vision Transformer (ViT)
**არქიტექტურა:** 8x8 patch embedding → positional encoding → 4 transformer encoder layer → CLS token კლასიფიკატორი

**გადაწყვეტილება:** გამოვცადე თანამედროვე transformer-ზე დაფუძნებული vision არქიტექტურა.

**შედეგი:** Val Acc: 29.5% (თითქმის შემთხვევითი გამოცნობა)

**ანალიზი:** ViT სრულად ჩაიჭრა ნულიდან ამ მონაცემთა set-ზე. ეს ViT-ების ცნობილი შეზღუდვაა:
- მათ სჭირდებათ დიდი მონაცემთა ნაკრებები (100k+ სურათი) ამის ნულიდან შესასწავლად
- FER2013-ის 23k სურათი ძალიან ცოტაა

**დასკვნა:** თანამედროვე არქიტექტურები ყოველთვის უკეთესი არ არის.

---

### 8. DeepCNNv2 (Residual Connections) საუკეთესო მოდელი
**არქიტექტურა:** 5 residual blocks (32→64→128→256→512 ფილტრი) + Global Average Pooling + AdamW + Cosine Annealing LR

**გადაწყვეტილება:** დავამატე residual connections უფრო ღრმა ქსელების gradient vanishing-ის გარეშე გასაწვრთნელად, AdamW-თან და cosine LR schedule-თან ერთად.

**შედეგი:** Train Acc: 69.8% | Val Acc: **66.1%** (ახალი საუკეთესო!)

**ანალიზი:**
- Residual connections-მა 5-block ქსელის ეფექტური სწავლება შესაძლო გახადა
- AdamW-ის სწორი weight decay-მ გენერალიზაცია გააუმჯობესა Adam-თან შედარებით
- Cosine annealing-მა უზრუნველყო LR-ის შეუფერხებელი შემცირება
- Train Acc (69.8%) > Val Acc (66.1%) — overfitting იწყება

## მთავარი დასკვნები

1. **სიღრმე + Residual Connections > მარტივი სიღრმე** — DeepCNNv2-მა residual connections-ით მნიშვნელოვნად გადააჭარბა DeepCNN-ს, რაც ადასტურებს residual connections-ის აუცილებლობას ძალიან ღრმა ქსელებისთვის.

2. **სწავლის სიჩქარე კრიტიკულია** — lr=0.01 სრულ დივერგენციას გამოიწვია Adam-თან, lr=0.001 კი ოპტიმალური აღმოჩნდა.

3. **სიმარტივე ზოგჯერ უკეთესია** — CNNLSTM-ის 1 layer-ი სჯობს 2 layer-ს. ViT ნულიდან სრულად ჩაიჭრა. სირთულის ზრდა ყოველთვის არ ეხმარება.

4. **Transfer Learning-ს დომეინების შესაბამისობა სჭირდება** — ResNet18 RGB ImageNet-ზე ვერ გადავიდა გრეისქეილ 48x48 სახის სურათებზე.

5. **BatchNorm + Dropout აუცილებელია** — MediumCNN-ის გაუმჯობესება SimpleCNN-თან (+7.7%) თითქმის მთლიანად ამ ორს უკავშირდება.

6. **Underfitting → Overfitting პროგრესია** — SimpleCNN underfitting-ს განიცდის (val > train), DeepCNNv2 ოდნავ overfitting-ს (train > val), რაც მოდელის სიმძლავრის ზრდასთან ერთად პროგრესს ასახავს.
